# 用户技术熟悉度档案

本文件用于帮助 Codex 根据用户对不同技术的熟悉程度调整解释深度。评分只反映当前对话和协作中可观察到的证据，不是能力鉴定；用户可随时纠正评分或要求删除记录。

## 评分标准

- 1–10，坚韧黑铁：刚开始接触，需要解释全部背景、术语和完整步骤。
- 11–20，英勇黄铜：具备少量基础认识，需要解释基础原理和操作原因。
- 21–30，不屈白银：能够理解常见概念，需要标准做法和具体示例。
- 31–40，荣耀黄金：具备基础实践能力，需要完整流程和常见错误说明。
- 41–50，华贵铂金：中等熟悉，需要解释核心机制、关键实现和坑点。
- 51–60，流光翡翠：能够独立处理常见问题，可直接讨论实现要点和主要取舍。
- 61–70，璀璨钻石：熟练处理实际问题，可聚焦复杂边界、排查和替代方案。
- 71–80，超凡大师：具备深入实践经验，可聚焦架构影响、性能安全和精准验证。
- 81–90，傲世宗师：可采用同行式精炼讨论，重点关注少见风险、反例和深层机制。
- 91–100，最强王者：极高熟悉度，只提供高信息密度的新结论、极端边界和证据。

置信度分为低、中、高。低置信度评分应保守使用，回答时仍需从当前问题持续校正。

## 回答深度映射

| 英雄联盟排位 | 回答方式 |
| --- | --- |
| 坚韧黑铁 | 解释全部背景、首次出现的术语、完整步骤和每一步目的 |
| 英勇黄铜 | 补充基础原理、操作原因和简单示例 |
| 不屈白银 | 给出标准做法、具体示例、常见概念和验证方法 |
| 荣耀黄金 | 说明完整流程、关键代码或命令以及常见错误 |
| 华贵铂金 | 聚焦核心机制、关键实现、坑点和验证方式 |
| 流光翡翠 | 先给结论，再讲实现要点、主要取舍和风险 |
| 璀璨钻石 | 聚焦复杂边界、排查路径和替代方案 |
| 超凡大师 | 聚焦架构影响、性能安全、精准验证和复杂决策 |
| 傲世宗师 | 使用同行式精炼表达，讨论少见风险、反例和深层机制 |
| 最强王者 | 只提供高信息密度的新结论、精确差异、极端边界、证据和可争论点 |

- 每个问题先识别主要技术类别和必要前置技术，再分别读取对应评分。
- 多技术问题按各部分评分分别调整深度，不使用综合评分或简单平均分。
- 若低分前置技术影响理解，先简短补齐前提；高分部分仍保持相应的信息密度。
- 当前问题本身也是校准信号，但不能脱离档案证据，仅凭一次提问大幅改变解释深度。

## 技术水平称呼

称呼统一使用：`技术主题 + 英雄联盟排位等级 + 当前分数`。

| 评分 | 英雄联盟排位 | 完整示例 |
| --- | --- | --- |
| 1–10 | 坚韧黑铁 | `Linux·坚韧黑铁（8 分）` |
| 11–20 | 英勇黄铜 | `Docker·英勇黄铜（18 分）` |
| 21–30 | 不屈白银 | `Git·不屈白银（28 分）` |
| 31–40 | 荣耀黄金 | `SQL·荣耀黄金（38 分）` |
| 41–50 | 华贵铂金 | `前端·华贵铂金（48 分）` |
| 51–60 | 流光翡翠 | `Spring·流光翡翠（58 分）` |
| 61–70 | 璀璨钻石 | `前端·璀璨钻石（66 分）` |
| 71–80 | 超凡大师 | `Java·超凡大师（78 分）` |
| 81–90 | 傲世宗师 | `架构·傲世宗师（88 分）` |
| 91–100 | 最强王者 | `AI/Codex·最强王者（92 分）` |

- 称呼应选择当前问题最相关的一项技术，每次回复最多使用一次。
- 英雄联盟排位和当前分数不可省略，不添加额外角色或修饰词。
- 低置信度标注“暂定”，例如 `前端·璀璨钻石（暂定，66 分）`。
- 正式、高风险、故障焦虑或用户不希望看到称呼的场景，可以不使用称呼。
- 用户明确指定、拒绝或纠正称呼时，以用户偏好为准。

## 更新约束

- 只依据用户的明确说明、实际操作、技术判断、问题深度、纠错和持续反馈更新相关类别。
- 普通有效证据通常调整 1–3 分，强证据最多调整 5 分；用户明确指定自身评分时优先采用。
- 不因询问基础问题、偶然失误、表达简短或缺少上下文而直接降低评分。
- 新技术类别按需新增，不为了覆盖“所有技术名词”预先创建大量无证据条目。
- 每次调整同步更新评分、置信度、依据和日期；最近调整记录只保留最新 20 条。
