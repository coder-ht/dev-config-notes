## 学习目标

### 技术依赖

- Linux 命令行基础：文件路径、标准输入、标准输出和标准错误。
- 本计划中的 Linux 依赖只作为前置知识记录，不计入 Shell 评分。

### 核心概念

- 变量赋值与展开：`NAME="kaiyun"`、`$NAME`、`$1`。
- 参数与安全引用：`"$@"`，以及路径、文件名和参数中包含空格时的处理。
- 条件判断：`if/else`、`-f`、`-d`、字符串和数字的基础比较。
- 命令结果：`$?`、退出状态码，以及根据命令成功或失败执行不同分支。
- 文件与管道：`>`、`>>`、`<`、`|`、`grep` 的常见组合。
- 通配符与循环：`*.log`、`for` 的基础用法，以及没有匹配文件时的常见现象。

### 标准做法

- 变量引用默认使用双引号，例如 `mkdir -p "$DIR"`。
- 判断命令是否成功时立即检查退出状态，避免被后续命令覆盖。
- 先确认文件或目录类型，再执行复制、读取或删除操作。
- 对日志筛选和结果输出使用清晰的管道、重定向，并保留必要的错误提示。

### 常见错误

- 使用 `$@` 而不是 `"$@"`，导致含空格参数被拆分。
- 使用未加引号的变量路径，导致一个路径被当成多个参数。
- 在读取 `$?` 前执行其他命令，导致检查到错误的退出状态。
- 把 `-f` 误认为“任意路径存在”，忽略它只匹配普通文件。
- 忽略通配符没有匹配文件时可能保留字面量的情况。

## 验证练习

- 写一个脚本，检查第一个参数是否为普通文件并输出结果。
- 写一个脚本，执行命令后根据 `$?` 输出成功或失败。
- 写一个脚本，安全处理带空格的目录参数。
- 遍历当前目录的 `.log` 文件并筛选包含 `ERROR` 的行。
- 解释一段包含变量、`if`、重定向、管道和 `for` 的短脚本。

## 条目完成条件

- 变量与安全引用：能解释变量展开和 `"$@"`，通过两道带空格参数的变式题，并独立写出安全参数循环。
- 条件判断与退出状态：能区分 `-f`、`-d` 和 `$?`，通过两道成功/失败分支题，并独立写出检查脚本。
- 重定向与管道：能区分 `>`、`>>`、`<` 和 `|`，通过一道结果预测题和一道日志处理题。
- 通配符与循环：能解释 `*.log` 展开和 `for`，通过两种文件列表场景，并独立遍历日志文件。
- 综合脚本：独立完成一个包含参数、判断、命令结果和日志输出的小脚本。
