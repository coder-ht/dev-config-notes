Shell 脚本学习文档

当前阶段：不屈白银（26 分）
学习目标：读懂并编写包含变量、判断、命令结果、日志处理和基础循环的小型 Shell 脚本。

一、变量与参数

变量

NAME="kaiyun"
echo "$NAME"
  • NAME="kaiyun"：给变量赋值。
  • $NAME：读取变量值。
  • "$NAME"：读取变量并保留空格。

等号两边不能有空格：NAME="kaiyun"正确，NAME = "kaiyun"错误。

位置参数

./script.sh frontend "release v1"
  • $0：脚本名称。
  • $1：第一个参数。
  • $2：第二个参数。

全部参数与安全引用

for arg in "$@"; do
  echo "$arg"
done

"$@"会保留每个参数的边界。路径、变量和参数通常应使用双引号。

二、条件判断

if [ -f "$FILE" ]; then
  echo "普通文件存在"
else
  echo "普通文件不存在"
fi
判断含义
-f普通文件
-d目录
-e路径存在
-z "$VALUE"字符串为空
-n "$VALUE"字符串非空
-f只判断普通文件，不代表任意路径存在。

三、命令执行结果

cp source.txt target.txt
echo $?

$?表示上一条命令的退出状态码。通常0表示成功，非0表示失败或异常。下一条命令会覆盖它。

cp source.txt target.txt
STATUS=$?
if [ "$STATUS" -eq 0 ]; then
  echo "复制成功"
else
  echo "复制失败"
fi

四、文件重定向

  • >：覆盖写入。
  • >>：追加写入。
  • <：输入重定向。
echo "hello" > output.txt
echo "world" >> output.txt
grep "ERROR" < app.log

echo默认会输出换行，因此连续写入通常会得到多行内容。

五、管道与日志筛选

cat app.log | grep "ERROR"
grep "ERROR" app.log > error.log

|把左侧命令的标准输出交给右侧命令作为输入。grep用于筛选匹配文本。

六、通配符与循环

for file in *.log; do
  echo "$file"
done

*.log表示当前目录下所有以.log结尾的文件。循环会依次处理匹配到的文件。文件名可能有空格，因此使用"$file"。

没有匹配文件时，*.log可能保留为字面量，需要注意这种情况。

七、基础脚本结构

#!/usr/bin/env bash

FILE="${1:-}"

if [ -z "$FILE" ]; then
  echo "请提供文件路径"
  exit 1
fi

if [ -f "$FILE" ]; then
  echo "普通文件存在：$FILE"
else
  echo "普通文件不存在：$FILE"
fi

这段脚本综合使用了位置参数、空值判断、文件判断和退出状态。

八、当前阶段常见错误

  1. 使用$@而不是"$@"。
  2. 路径变量不加双引号。
  3. 把$?当成命令输出。
  4. 执行其他命令后才读取$?。
  5. 把-f当成任意路径存在判断。
  6. 混淆>、>>和管道。
  7. 不清楚*.log会匹配哪些文件。
  8. 循环中不引用文件变量。

九、自测方向

  • 解释$@与"$@"的区别。
  • 预测带空格目录变量的实际执行结果。
  • 解释$?的含义及读取时机。
  • 区分-f、-d和-e。
  • 预测>和>>执行后的文件内容。
  • 解释*.log和for的循环次数。
  • 独立编写一个检查文件并筛选日志的短脚本。
