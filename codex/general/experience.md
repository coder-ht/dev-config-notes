# 通用经验总结

本文档记录所有环境都适用的 Codex 协作、编码、规则维护和仓库操作经验。

## 经验归属规则

- 所有环境都适用的经验写入本文件。
- 只和某个环境的路径、项目、配置、账号、机器状态相关的经验，写入对应环境目录下的 `experience.md`。
- 不确定归属时，先询问用户，不要随意写入通用经验。
- 经验总结不得包含 Bearer、账号密码、API Key、测试环境长期凭据、个人敏感信息或运行日志。

## 2026-07-09 Codex 规则迁移与路由

- 场景：需要把 Codex 全局规则放入仓库，方便不同机器的新 Codex 复用。
- 做法：`AGENTS.md` 只保留环境路由逻辑，具体规则拆到 `codex/general/` 与 `codex/envs/<env>/`。
- 复用：任务开始时先读取 `CODEX_WORK_ENV`，始终加载通用规则/经验，再按环境加载对应规则/经验。
- 注意：写入经验前必须判断归属，通用经验写入 `codex/general/experience.md`，环境专属经验写入对应环境目录。

## 2026-07-09 Codex 规则同步与备份术语

- 场景：用户要求区分“同步 Codex 规则”和“备份 Codex 规则”的方向。
- 做法：“同步”固定表示规则仓库到本机 Codex 配置；“备份”固定表示本机 Codex 配置到规则仓库，并在备份后提交推送。
- 注意：执行前必须确认方向和 `CODEX_WORK_ENV`，并排除 Bearer、账号密码、API Key、测试环境长期凭据、个人敏感信息和运行日志。

## 2026-07-09 Codex 规则同步前拉取仓库

- 场景：把规则仓库内容同步到本机 Codex 配置前，仓库可能落后于远端。
- 做法：同步前先在规则仓库根目录拉取最新代码，优先使用 `git pull --ff-only`；无法快进或拉取失败时停止同步。
- 注意：规则仓库存在未提交改动时，先说明改动范围，避免用旧内容或冲突状态覆盖本机规则。

## 2026-07-09 Codex 环境变量强制要求

- 场景：用户要求 Codex 必须配置环境变量后才能正常工作。
- 做法：`CODEX_WORK_ENV` 是必需环境变量，当前支持 `ghy` 和 `home`；缺失或未知时必须停止具体任务并提示用户配置。
- 注意：规则维护任务可用于落地这条约束；后续普通任务、同步、备份、提交、推送等都必须先确认 `CODEX_WORK_ENV` 有效。

## 2026-07-09 Codex 执行说明与经验参考强化

- 场景：用户要求加强经验总结、经验参考、执行前计划说明和执行后说明的规则约束。
- 做法：在通用规则中把经验检索、计划内容、执行后说明、经验总结写入拆成明确检查项，要求说明命中经验或未发现相似经验，并区分已完成改动、验证结果、未验证原因和遗留事项。
- 注意：经验只能辅助缩小范围，不能替代读取当前真实文件、命令输出、配置或接口响应；经验总结应短小可复用，不写一次性流水账或敏感信息。
