# Kaihe 项目经验总结

## 需求到代码追踪

- 场景：分析业务需求、原型并追踪到代码或数据结构。
- 做法：遵循“需求文档 -> 原型入口/页面 -> 当前项目代码 -> 数据结构”，从需求确认规则，从原型确认交互，再回到实现验证。
- 注意：页面字段不能单独依据截图或单个 HTML 推断；需要结合入口导航、原型页面和需求文档。

## 原型问题排查

- 场景：页面显示、滚动、iframe 或入口流程异常。
- 做法：先确认真实打开路径，再追踪 iframe、导航和具体业务页面，确认真正拥有事件和滚动的容器。
- 注意：原型页面单独打开正常，不代表实际产品入口正常。

## 内容真实性

- 场景：需求文档、AI 结论、法规引用或业务数据说明。
- 做法：保留来源入口，标注版本和推断边界；法规绑定条款，量化数据绑定可追溯来源。
- 注意：Kaihe 的角色、页面、业务语义和文档结构不能直接当作其他项目的事实。
